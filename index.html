System Design

Architecture Overview

A serverless, no-code/low-code architecture leveraging LINE, Make, Google Sheets, and AI services.

LINE OA

User sends message (text/voice/photo)

→

Webhook / Make

Receives & routes to processing flow

→

Google Sheets DB

Stores all transactions & user data

→

AI Categorizer

Classifies transactions via GPT/Gemini

→

Reply (Flex Msg)

Sends formatted response to user

Why this architecture?

  • Zero server costs — Google Sheets + Make free tier handles typical personal usage easily
  • Familiar interface — Users interact via LINE, no new app to install
  • Easy debugging — All data visible in spreadsheet; Make shows execution logs
  • Extensible — Can add new automations, integrations, or migrate to a real DB later
FAQ

Frequently Asked Questions

Is my financial data secure?

This demo stores data only in your browser's localStorage. The actual LINE OA system uses Google Sheets with restricted access (only the owner can view). No data is shared with third parties. For production, consider encrypting sensitive fields and implementing proper access controls.

How does the system handle multiple users?

Each LINE user has a unique LineUserId. The system stores and filters all records by this ID, ensuring users only see their own data. The Google Sheet has one row per transaction with the user ID as a column, enabling multi-tenant data storage.

What are the API costs for AI categorization?

Using GPT-4o-mini or Gemini Flash, typical costs are under $0.01 per 100 transactions. For a personal finance app with ~50 transactions/month, expect less than $0.10/month. The free tier of most AI APIs covers this comfortably.

Why separate chat mode from logging mode?

Without mode separation, every message would be parsed as a potential transaction, leading to false positives (e.g., "I ate 50 baht worth of food" might log incorrectly). Dedicated modes let users switch context: logging mode expects structured input, chat mode enables free conversation with the AI assistant.

How is data backed up?

Google Sheets provides automatic version history and can sync to Google Drive. For additional safety, the Make automation can be extended to create weekly backups to a separate Sheet or export to CSV. The LINE OA side has no persistent state—all data lives in the Sheet.

What future features are planned?

Potential enhancements include: bill reminders, recurring transaction detection, investment tracking, multi-currency support, PDF export of monthly reports, and integration with bank APIs (where available). The modular architecture makes adding features straightforward.
